💡 Quick Decision Guide

✅ Use O(1) - Constant

  • Array index access
  • Hash map lookup
  • Stack push/pop
  • Simple calculations

✅ Use O(log n) - Logarithmic

  • Binary search in sorted array
  • Balanced tree operations
  • Divide and conquer

✅ Use O(n) - Linear

  • Single loop through data
  • Linear search
  • Array traversal
  • Most array operations

✅ Use O(n log n) - Linearithmic

  • Efficient sorting (Merge, Quick)
  • Divide and conquer with merge
  • Best we can do for sorting

❌ Avoid O(n²) - Quadratic

  • Nested loops
  • Bubble/Selection sort
  • Comparing all pairs
  • Only for small data (<100 items)

💀 Never Use O(2ⁿ) - Exponential

  • Naive recursive Fibonacci
  • Brute force solutions
  • Trying all subsets
  • Computer will freeze!

🎓 Interview Tips

When Interviewer Asks "What's the time complexity?"

  • They mean: Big-O (worst case)
  • Say: "O(n)" not "Θ(n)" or "Ω(n)"
  • Explain: "In the worst case, we check all n elements, so O(n)"
  • Bonus: Mention space complexity too: "O(n) time, O(1) space"
